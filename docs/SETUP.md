# セットアップ手順書 — Cloud SQL (MySQL)

Phase 0 で必要な、GCP課金設定〜Cloud SQL作成〜GAS接続確認までの手順です。
所要時間の目安: 30〜45分（インスタンス作成の待ち時間を含む）。

## 1. GCPプロジェクトの準備

1. https://console.cloud.google.com を開き、会社のGoogleアカウントでログイン
2. 画面上部のプロジェクト選択 → 「新しいプロジェクト」
   - プロジェクト名: `corp-apps`（例。任意の名前でOK）
3. 左メニュー「お支払い」→ 課金アカウントを作成し、プロジェクトに紐付け
   - クレジットカード登録が必要です
   - 初めての場合、無料トライアルクレジット（$300/90日）が付く場合があります

## 2. Cloud SQL インスタンスの作成

1. 左メニュー「SQL」→「インスタンスを作成」→「MySQL を選択」
   - 初回は「Compute Engine API の有効化」を求められたら有効化する
2. 設定値:
   - **インスタンスID**: `corp-shared-db`
   - **rootパスワード**: 強固なものを生成してパスワードマネージャ等に保管
   - **データベースバージョン**: MySQL 8.0
   - **Cloud SQLのエディション**: Enterprise
   - **プリセット**: 「サンドボックス」または「開発」（最小構成。月額 約$9〜15）
   - **リージョン**: `asia-northeast1`（東京）
   - **ゾーンの可用性**: シングルゾーン
3. 「接続」設定（インスタンス作成画面の「構成オプションを表示」内、または作成後に編集）:
   - **パブリックIP**: 有効のまま
   - **承認済みネットワーク**: 追加不要（GASの`Jdbc.getCloudSqlConnection`はGoogle内部経路で接続するため）
4. 「インスタンスを作成」→ 完成まで5〜10分待つ

## 3. データベースとユーザーの作成

1. 作成したインスタンスを開く →「データベース」タブ →「データベースを作成」
   - 名前: `corp_shared`（文字セット: utf8mb4）
2. 「ユーザー」タブ →「ユーザーアカウントを追加」
   - ユーザー名: `gas_app`
   - パスワード: 強固なものを生成して保管（rootとは別にする）
   - ホスト名: すべてのホストを許可（%）

## 4. スキーマの適用

1. インスタンス画面 →「Cloud SQL Studio」→ データベース `corp_shared` にrootでログイン
2. このリポジトリの `sql/schema.sql` の中身をエディタに貼り付けて実行
3. 左のツリーにテーブル6つ（employees, receipts, approval_routes, approval_steps, freee_uploads, app_logs）が表示されればOK

※ Cloud SQL StudioはそのままSQL学習の練習場としても使えます（SELECT/INSERT等を自由に試せます）。

## 5. GAS への接続情報設定

1. GASエディタ（このプロジェクトの `gas/` を clasp push したもの）を開く
2. 左メニュー「プロジェクトの設定」→「スクリプト プロパティ」→ 以下4つを追加:

| プロパティ名 | 値 |
|---|---|
| `DB_INSTANCE_CONNECTION_NAME` | インスタンスの「概要」ページの**接続名**（例: `corp-apps:asia-northeast1:corp-shared-db`） |
| `DB_NAME` | `corp_shared` |
| `DB_USER` | `gas_app` |
| `DB_PASS` | 手順3で作成したパスワード |

## 6. 接続テスト（E2E確認）

1. GASエディタで関数選択ドロップダウンから `testDbConnection` を選択して ▶ 実行
2. 初回は権限承認ダイアログが出るので許可する
3. 実行ログに以下が出れば成功:
   - `✅ 接続成功`
   - `✅ テーブル一覧`（6テーブル）
   - `✅ ログ書き込み成功`（app_logsに1行追加される）

## トラブルシューティング

- **「DB接続情報が未設定です」** → 手順5のスクリプトプロパティを確認
- **接続タイムアウト・Failed to establish a database connection** → インスタンスの「パブリックIP」が有効か確認。GCPプロジェクトの課金が有効か確認
- **Access denied for user** → DB_USER/DB_PASSの値、ユーザーのホスト設定（%）を確認
